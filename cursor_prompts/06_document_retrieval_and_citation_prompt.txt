Primary goal:
Enable authoritative retrieval from the approved local corpus, with explicit citation objects.

Functional requirements:
- search only within the locally ingested approved corpus
- support at least a lexical search path first
- optionally add a simple semantic retrieval path only if it remains fully local and maintainable
- allow filtering by:
  - active project standard family
  - language
  - document_id
  - topic or tag if available
- return ranked results with explicit citation payloads
- define a citation object that can later be shown in UI and reports, including at minimum:
  - document title
  - document_id
  - chunk_id
  - page/location if available

- no answer object should imply authority without a citation payload
- if the corpus is insufficient, the system must say so structurally
- retrieval must not silently mix inactive standards if the project restricts them
